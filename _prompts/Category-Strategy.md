<concept_disambiguation>
# Key Distinctions  
• Category Management = end-to-end governance model (people, process, technology).  
• Category Strategy = the playbook (analysis → plan → execution) produced for ONE category.  
• “Researchable” fields = Any element that can be answered via reputable public sources, databases or
  benchmark subscriptions.  
• “Company-specific” fields = Internal spend, contracts, KPIs, policy or stakeholder data that is NOT
  discoverable externally.  These will be left as yellow-highlighted placeholders for the user to fill in.
</concept_disambiguation>

<frameworks>
# Framework Library

Below are standardized frameworks to guide content creation for every subsection. AI-generated sections include a concise *Mini-Prompt* to steer automatic drafting; client sections omit the prompt but provide the structure so users can supply company-specific inputs.

## 1. Overview, Definitions and Orientation
1.1 Document Overview — Executive summary template (purpose, audience, scope, document map).
1.2 What is Category Management? — Framework: CIPS / ISM definition with value-chain orientation. *Mini-Prompt:* “Define category management in 120 words citing CIPS and ISM.”
1.3 Components of Category Strategy — Framework: Analyse → Plan → Execute cycle with KPI linkage. *Mini-Prompt:* “List and briefly describe the core components of a category strategy in ≤150 words.”
1.4 Category Strategy Source List — Framework: Annotated bibliography template (Source, Type, Coverage, URL). *Mini-Prompt:* “Generate a table of 10 authoritative sources (analyst, gov, academic, trade), include URL.”

## 2. Category Strategy Outline
### 2.1 Category Profile
2.1.1 Category Definition & Taxonomy — Framework: UNSPSC / NAICS hierarchy (Segment → Family → Class). *Mini-Prompt:* “Provide a formal definition and taxonomy levels for the category (≤150 words).”
2.1.2 Category Scope — Framework: Scope matrix (Included / Excluded spend categories+geos).

### 2.2 Stakeholder Management & Governance
2.2.1 Stakeholder Identification — Framework: Stakeholder map (Power vs Interest grid).
2.2.2 Defining Roles and Responsibilities — Framework: RACI matrix (Role, Accountabilities). *Mini-Prompt:* “Draft a RACI table for five key stakeholder roles.”
2.2.3 Governance & Engagement Plan — Framework: Engagement ladder (Inform → Consult → Involve → Collaborate → Empower).

### 2.3 Internal Analysis
2.3.1 Spend Analysis — Framework: Pareto chart of spend by supplier / sub-category.
2.3.2 Vendor Profiling — Framework: Supplier scorecard (Spend, Criticality, Performance).
2.3.3 Contract Analysis — Framework: Contract repository template (Value, Term, Renewal).
2.3.4 Payment Terms Analysis — Framework: DPO benchmarking table (Supplier, Terms, Variance).
2.3.5 Category Maturity Assessment — Framework: 5-Level model (Ad-hoc → World-Class) with descriptors. *Mini-Prompt:* “Describe the five maturity levels and criteria in a table.”
2.3.6 SWOT Analysis — Framework: 2×2 SWOT grid. *Mini-Prompt:* “Populate a SWOT grid with ≤6 bullets per quadrant.”
2.3.7 Category Segmentation — Framework: Kraljic matrix quadrants. *Mini-Prompt:* “Assign sub-segments to Kraljic quadrants and explain rationale (≤150 words).”
2.3.8 Category Segmentation — Framework: Segmentation summary table (Strategic, Leverage, etc.).

### 2.4 External Analysis
2.4.1 Market Overview & Trends — Framework: TAM, SAM, CAGR, key drivers. *Mini-Prompt:* “Summarize market size, CAGR, and top 3 trends with citations (≤200 words).”
2.4.2 Vendor Landscape — Framework: Top-supplier table with market share %. *Mini-Prompt:* “List leading suppliers with estimated market shares in a table.”
2.4.3 Industry Cost Structure — Framework: Cost driver breakdown pie chart data. *Mini-Prompt:* “Provide cost structure percentages suitable for a pie chart.”
2.4.4 PESTLE Analysis — Framework: PESTLE table (Political→Legal). *Mini-Prompt:* “Fill a PESTLE table with 2 factors per dimension.”
2.4.5 Porter’s Five Forces Analysis — Framework: Force rating table (High/Med/Low, rationale). *Mini-Prompt:* “Rate each force and justify in one sentence.”
2.4.6 Risk Analysis — Framework: Risk register (Risk, Probability, Impact, Mitigation). *Mini-Prompt:* “Create a 5-row risk register with probability & mitigation.”
2.4.7 Recent News & Market Intelligence — Framework: News digest (Date, Source, Headline, Implication). *Mini-Prompt:* “Provide 3 recent (<24 months) news items and their implications.”

### 2.5 Formulating Category Strategy
2.5.1 Business Requirements — Framework: Requirement tree (Need → KPI → Owner).
2.5.2 Category Goals — Framework: SMART goals table (Goal, Metric, Baseline, Target, Timing).
2.5.3 Opportunity Assessment & Strategy Plan — Framework: Benefit vs Effort matrix. *Mini-Prompt:* “List 5 opportunities ranked by benefit and effort in a table.”
2.5.4 Identify Initiatives — Framework: Initiative pipeline (Initiative, Owner, ROI, Status).
2.5.5 Sourcing Pipeline — Framework: Prioritized initiative roadmap (Quarter, Initiative, Value).

### 2.6 Execution Plan
2.6.1 Establish Execution Timeline — Framework: Gantt-style timeline (Milestone, Start, End).
2.6.2 Detailed Execution Plan — Framework: Work-breakdown structure (Task, Responsible, Duration).
2.6.3 Savings Tracking Template — Framework: Savings tracker (Initiative, Baseline, Forecast, Realized, Timing). *Mini-Prompt:* “Provide a savings tracker template with example line.”
2.6.4 Maintenance & Revision — Framework: Governance cadence timeline (Quarterly review, Annual refresh). *Mini-Prompt:* “Outline maintenance activities across four quarters.”
</frameworks>
